Uso de Estructuras de Control

2. Estructura secuencial: sentencias y bloques

Conceptos básicos a recordar:

  • ¿Cómo se escribe un programa? Colocando las instrucciones en el orden de ejecución deseado.
  • ¿Una línea por sentencia? Sí, es muy recomendable para facilitar la lectura y localización de errores.
  • ¿Punto y coma al final? Sí, todas las sentencias terminan con ; (excepto las cabeceras de estructuras de control).
  • ¿Qué es un bloque? Conjunto de sentencias entre llaves { } que se ejecuta como una unidad. Se usa en estructuras de control, clases y métodos.

Importante: Documenta tu código con claridad, nombres descriptivos y sin faltas de ortografía (tanto en comentarios como en cadenas literales).

2.1. Ámbito o alcance de una variable

El ámbito de una variable es la zona donde "existe". Está definido por el bloque (entre llaves) donde se declaró. Al cerrar la llave }, la variable desaparece.

Ejemplo de ámbito de variables Java 
public static void main(String[] args) {
    int num1 = 10; // Existe en todo el main
    {
        int num2 = 20; // Solo existe en este bloque
        num1++; // Puede usar num1
    }
    // num2 ya NO existe aquí - daría error
}
